1. What is a VCF File?

Variant Call Format (VCF) is a standardized text file format used in bioinformatics for storing genetic variation data. It contains meta-information lines, a header line, and data lines describing genetic variants (SNPs, indels, structural variants) and their annotations.

2. Generating VCF Files

2.1 Single Sample SNP Calling

# Single sample SNP calling using bcftools bcftools mpileup -Ou -f reference.fasta sample.sorted.bam --threads 8 \ --annotate INFO/AD,FORMAT/DP,FORMAT/AD \ | bcftools call -Ou -mv \ | bcftools +fill-tags -- -t AC,AN,AF,MAF,HWE,ExcHet \ | bcftools view -Oz > sample.raw.vcf.gz # Index the VCF bcftools index sample.raw.vcf.gz

2.2 Multi-Sample SNP Calling with BAM List

📋 BAM List Format: Create a text file listing all BAM files (one per line) for joint calling. This enables simultaneous variant calling across multiple samples, which is essential for population genetics.
# Example bamlist.txt content: # /path/to/sample1.sorted.bam # /path/to/sample2.sorted.bam # /path/to/sample3.sorted.bam

2.3 Complete Multi-Sample SNP Calling Pipeline

function snp_calling { BAMLIST=$1 OUTDIR=$2 REF=$3 PREFIX=$4 # FILTERING PARAMETERS min_base_mapping_quality=10 # Minimum base mapping quality minQ=20 # Minimum variant quality score MIN_DP=7 # Minimum total depth MIN_ALT_COUNT=1 # Minimum alternate allele count MIN_FMT_DP=5 # Minimum per-sample depth MIN_MINOR_ALLELE_FREQ=0.01 # Minimum minor allele frequency MIN_MAPPING_QUALITY=30 # Minimum mapping quality SIG=0.05 # Significance threshold for HWE echo 'Calling SNPs for' $PREFIX # Prepare sample lists tmpbam=tmp_bamlist.txt sample_rename=sample_rename.txt # Extract BAM paths and sample names tail -n +2 $BAMLIST | cut -d ',' -f 1 > $tmpbam tail -n +2 $BAMLIST | cut -d ',' -f 1 | rev | cut -d '/' -f 1 | rev | cut -d '.' -f 1 | rev | cut -d '-' -f 1 | rev > $sample_rename # Step 1: Initial variant calling bcftools mpileup -Ou -f $REF --bam-list $tmpbam --threads 80 \ --annotate INFO/AD,FORMAT/DP,FORMAT/AD \ | bcftools call -Ou -mv \ | bcftools +fill-tags -- -t AC,AN,AF,MAF,HWE,ExcHet \ | bcftools view -Oz > ${OUTDIR}/${PREFIX}.tmp.vcf.gz # Step 2: Rename samples bcftools reheader -s $sample_rename ${OUTDIR}/${PREFIX}.tmp.vcf.gz \ | bcftools view -Oz -o ${OUTDIR}/${PREFIX}.raw.vcf.gz # Step 3: Apply filters bcftools view ${OUTDIR}/${PREFIX}.raw.vcf.gz -O u 2>/dev/null \ | bcftools view --types snps \ | bcftools view -m2 -M2 \ | bcftools filter -e "QUAL<${minQ} || INFO/DP < $MIN_DP" \ | bcftools filter -i "INFO/AC[1] >= $MIN_ALT_COUNT" \ | bcftools filter -i "COUNT(FMT/DP<${MIN_FMT_DP}) <= 2" \ | bcftools filter -e "INFO/MAF[0] < $MIN_MINOR_ALLELE_FREQ" \ | bcftools filter -e "INFO/MQ < $MIN_MAPPING_QUALITY" \ | bcftools filter -e "HWE < $SIG" \ | bcftools filter -e "INFO/ExcHet < $SIG" \ | bcftools view -Oz -o $OUTDIR/${PREFIX}.flt3.vcf.gz # Index files bcftools index $OUTDIR/${PREFIX}.flt3.vcf.gz -f bcftools index $OUTDIR/${PREFIX}.raw.vcf.gz -f # Clean up rm ${OUTDIR}/${PREFIX}.tmp.vcf.gz }

3. Understanding VCF Format

3.1 Header Section (Lines starting with ##)

the VCF header section contains filtering information etc
🔍 Key Header Lines:
  • ##fileformat=VCFv4.2: Specifies the VCF version
  • ##INFO=...: Defines INFO fields (e.g., AC, AF, MAF)
  • ##FORMAT=...: Defines FORMAT fields for genotype data
    • This is the best place to know what parameters are available for filtering and analysis. You can also find the meaning of each annotation in the header, which is crucial for interpreting your data correctly. Always check the header to understand the annotations present in your VCF file.

3.2 Data Line Components

Column Field Description Example
1 CHROM Chromosome or contig name chr1
2 POS 1-based position 123456
3 ID Variant identifier (dbSNP rsID, etc.) rs123456
4 REF Reference allele A
5 ALT Alternate allele(s) G
6 QUAL Phred-scaled quality score 99.5
7 FILTER Filter status (PASS or failed filters) PASS
8 INFO Variant-level annotations AC=5;AF=0.125;MAF=0.125
9 FORMAT Format of genotype fields GT:DP:AD
10+ SAMPLES Per-sample genotype data 0/1:15:7,8

4. Key VCF Annotations and Their Population Genetics Significance

4.1 Allele-Based Statistics

AC (Allele Count): Count of alternate alleles in called genotypes
Population relevance: Used to calculate allele frequencies and for Fst estimation
AN (Allele Number): Total number of alleles in called genotypes
Population relevance: Essential for frequency calculations (AF = AC/AN)
AF (Allele Frequency): Frequency of each alternate allele
Population relevance: Core parameter for population genetics analyses
MAF (Minor Allele Frequency): Frequency of the less common allele
Population relevance: Used to identify rare variants and for demographic inference

4.2 Hardy-Weinberg Equilibrium Statistics

HWE (Hardy-Weinberg Equilibrium p-value): Probability that observed genotypes follow HWE
Population relevance:
  • p > 0.05: Population in HWE (random mating, no selection)
  • p < 0.05: Deviation from HWE - possible selection, inbreeding, or genotyping errors
  • Used to detect problematic markers or interesting evolutionary processes
ExcHet (Excess Heterozygosity): Measure "Probability of excess heterozygosity"
  • INFO/ExcHet Number:A Type:Float
  • Test excess heterozygosity; 1=good, 0=bad
  • basically, lower P values indicate potential excess heterozygosity

5. Filtering Strategies for Population Genomics

Filter Parameter Population Relevance
QUAL < minQ minQ=20 Remove low-confidence variants that could bias frequency estimates
INFO/DP < MIN_DP MIN_DP=7 Insufficient coverage can lead to inaccurate genotype calls and frequency estimates
INFO/AC[1] >= MIN_ALT_COUNT MIN_ALT_COUNT=1 Ensure variants are observed in at least one sample (remove monomorphic sites)
COUNT(FMT/DP MIN_FMT_DP=5 Allow up to 2 samples with low depth - balances data retention and quality
INFO/MAF[0] < MIN_MINOR_ALLELE_FREQ MIN_MAF=0.01 Remove rare variants that may be sequencing errors or private mutations
INFO/MQ < MIN_MAPPING_QUALITY MIN_MQ=30 Remove variants from repetitive regions that could cause mapping artifacts
HWE < SIG SIG=0.05 Remove markers deviating from HWE (potential genotyping errors or selection)
INFO/ExcHet < SIG SIG=0.05 Remove markers with significant excess heterozygosity (possible paralogs)

6. Population Genomics Analysis with vcflib

🔧 vcflib: A powerful C++ library for manipulating VCF files, providing numerous tools for population genetics analyses. GitHub Repository

6.1 Installing vcflib

# Clone and install vcflib git clone --recursive https://github.com/vcflib/vcflib.git cd vcflib make # Add to PATH export PATH=$PATH:/path/to/vcflib/bin

6.2 Basic vcflib Commands

vcfstats - Comprehensive Variant Statistics

# Generate comprehensive statistics from VCF vcfstats filtered.vcf.gz > stats.txt # Output includes: # - Number of SNPs, indels # - Transition/Transversion ratio # - Allele frequency distribution # - Genotype quality distribution

vcfallelicprimitives - Decompose Complex Variants

# Break down complex variants into primitive alleles # Essential for accurate population genetics vcfallelicprimitives filtered.vcf.gz | bgzip > decomposed.vcf.gz

vcfintersect - Compare VCF Files

# Find common variants between populations vcfintersect -a population1.vcf.gz -b population2.vcf.gz > shared.vcf.gz

6.3 Population Differentiation Analysis (Fst)

📊 Fst (Fixation Index): Measures genetic differentiation between populations. Ranges from 0 (no differentiation) to 1 (complete differentiation).
# Calculate Fst between two populations using vcflib # First, create population files echo "sample1 sample2 sample3" > pop1.txt echo "sample4 sample5 sample6" > pop2.txt # Use vcffixup to add allele frequencies vcffixup filtered.vcf.gz | vcffilter -f "AC > 0" | \ vcftools --vcf - --weir-fst-pop pop1.txt --weir-fst-pop pop2.txt --out population_comparison # Alternative: Use vcflib's vcfstats for per-site Fst # Note: You may need to use additional tools like vcftools for Fst calculation

8. Best Practices and Common Pitfalls

✅ Best Practices

  • Always inspect raw VCFs: Before filtering, understand your data distribution
  • Document filtering parameters: Keep track of thresholds for reproducibility
  • Use appropriate MAF thresholds: 0.01-0.05 for population studies, higher for selection scans
  • Consider linkage disequilibrium: Prune variants for many population genetics analyses
  • Validate with multiple methods: Cross-check results using different tools

⚠️ Common Pitfalls

  • Over-filtering: Removing too many variants can bias population genetics inferences
  • Ignoring missing data: High missingness can affect frequency estimates
  • Population structure: Undetected structure can violate HWE assumptions
  • Reference bias: Mapping to a reference can introduce biases in variant calling
  • Sample size effects: Small sample sizes lead to unstable frequency estimates
